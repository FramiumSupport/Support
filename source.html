Source Code Structure

📁 FRAMIUM/
📁 config/
📄 c2me.toml - Optimized chunk loading settings
📄 dynamicfps.json - Dynamic FPS settings when the window is inactive
📄 entityculling.properties - Entity rendering optimization
📄 ferritecore.properties - RAM usage reduction
📄 iris.properties - Shader configuration (if used)
📄 krypton.properties - Network optimization for multiplayer
📄 lithium.properties - General performance improvements
📄 sodium-options.json - Advanced rendering settings
📁 mods/
📦 (.jar files) - All optimization mods
📄 modlist.json - Full list of mods with versions
🖼️ icon.png - Modpack icon

Purpose of Each File

c2me.toml

Controls parallel chunk loading to improve performance in large worlds.

entityculling.properties

Skips rendering of invisible entities, reducing GPU usage.

ferritecore.properties

Optimizes memory by reducing redundant data.

krypton.properties

Improves multiplayer connection by compressing network packets.

lithium.properties

Speeds up game calculations (physics, AI, etc.) for better FPS.

sodium-options.json

Advanced configuration for the graphics renderer.
